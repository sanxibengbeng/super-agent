# Super Agent 产品使用手册

---

## 一、产品简介

Super Agent 是一个企业级多智能体平台，帮助企业将业务知识沉淀为标准化 SOP，再从 SOP 中孵化出能自主执行任务的虚拟员工（AI Agent）。通过可视化工作流将多个智能体串联协作，企业可以像搭积木一样构建自动化业务流程。

核心理念：**定义业务 → 创造智能体 → 固化工作流 → 持续进化**

---

## 二、登录与账号

### 2.1 首次登录

部署完成后，使用管理员在部署时设置的邮箱和密码登录系统。系统使用 AWS Cognito 进行身份认证，登录流程基于 OAuth 2.0 + PKCE 标准。

访问地址为部署时分配的 Elastic IP 或自定义域名，例如 `https://54.x.x.x`。

### 2.2 界面语言

系统支持中文和英文两种界面语言。点击左下角头像打开管理菜单，选择「语言切换」即可在中英文之间切换。

### 2.3 外观主题

进入「Settings → Appearance」可以切换浅色、深色或跟随系统的主题模式。

## 四、核心概念

### 4.1 组织（Organization）

组织是 Super Agent 的顶层租户单元。所有数据（智能体、工作流、对话、文档等）严格按组织隔离。每个用户属于一个组织，组织内有不同角色。

### 4.2 角色与权限

| 角色 | 权限 |
| --- | --- |
| Owner | 完整的组织管理权限，可管理成员、修改组织设置 |
| Admin | 可管理智能体、工作流、业务范围和各项配置 |
| Member | 可使用智能体和工作流，配置权限受限 |

### 4.3 业务范围（Business Scope）

业务范围是 Super Agent 的核心组织单元，用于按业务领域划分独立环境。例如：销售、人力资源、IT 运维、市场营销等。

每个业务范围拥有独立的：
- 智能体团队
- 知识库
- 技能包
- MCP 工具连接
- 工作流
- 记忆体
- IM 渠道绑定

业务范围支持设置可见性（开放/受限），并可通过 Scope Membership 进行细粒度的访问控制。

### 4.4 智能体（Agent）

智能体是 Super Agent 中的虚拟员工。每个智能体拥有：
- 名称和显示名称
- 角色定义
- 系统提示词（System Prompt）
- 模型配置参数
- 技能组合
- 所属业务范围

智能体可以独立工作（通过 Chat 对话），也可以在工作流中与其他智能体协作。

## 五、Dashboard（总览）

Dashboard 是登录后的首页，以业务范围为维度展示整个组织的智能体概况。

页面内容包括：
- 按业务范围分组的智能体卡片，每张卡片显示智能体名称、角色、状态和关键指标
- 系统统计数据：活跃智能体数量、已自动化任务数、活跃任务数
- 点击任意智能体卡片可快速进入对话或配置页面

管理员可以在 Dashboard 上直接创建新的业务范围。

## 六、Business Scope 管理

### 6.1 创建业务范围

有两种方式创建业务范围：

**手动创建**：点击 Dashboard 上的「创建业务范围」按钮，填写名称、描述、图标和颜色。

**AI 辅助创建**：选择「AI 生成」模式，用自然语言描述你的业务领域，AI 会自动生成业务范围的结构建议，包括推荐的智能体角色、技能和知识库配置。

### 6.2 业务范围配置

进入业务范围后，可以管理以下内容：

- **智能体**：创建和配置属于该范围的智能体
- **知识库**：上传文档，建立 RAG 知识检索能力
- **技能**：安装和管理该范围可用的技能包
- **MCP 工具**：配置外部工具连接器
- **记忆体**：查看和管理范围级别的持久化知识
- **IM 渠道**：绑定 Slack、钉钉、飞书等即时通讯渠道
- **简报**：查看 AI 自动生成的业务范围简报
- **成员**：管理范围级别的访问权限

### 6.3 Scope Memory（记忆体）

记忆体是业务范围级别的持久化知识库。智能体在执行任务过程中积累的经验、教训和洞察会自动记录为记忆条目。

记忆条目支持：
- 分类标签（lessons、insights 等）
- 置顶标记
- 自定义标签
- 手动创建和编辑

记忆体让智能体越用越聪明，新创建的智能体也能继承范围内已有的经验。

### 6.4 Scope Briefing（智能简报）

系统会定期自动生成业务范围简报，汇总以下信息：
- 工作流执行情况
- 对话会话摘要
- 文档处理状态
- 新增记忆条目
- 重要性评分

简报支持已读/归档状态管理。

## 七、Agent 管理

### 7.1 创建智能体

在「Agents」页面或业务范围内点击创建按钮，填写以下信息：

- **名称**：智能体的唯一标识名（英文，用于系统内部引用）
- **显示名称**：用户可见的友好名称
- **角色**：智能体的职能描述，如「销售助理」「HR 专员」
- **系统提示词**：定义智能体的行为准则和专业知识
- **所属业务范围**：将智能体分配到特定的业务领域
- **模型配置**：调整 AI 模型参数

### 7.2 Agent Configurator（智能体配置器）

点击智能体卡片进入详细配置页面，可以：

- 编辑基本信息和系统提示词
- 查看和管理已装备的技能
- 查看智能体活动指标和事件日志
- 生成或上传头像

### 7.3 Skill Workshop（技能工坊）

在智能体配置页面进入技能工坊，可以：

- 浏览可用技能列表
- 为智能体装备（Equip）或卸载（Unequip）技能
- 测试技能效果
- 查看技能详情和使用说明

## 八、Chat（对话）

Chat 是与智能体交互的核心界面。

### 8.1 发起对话

进入 Chat 页面后，有两种对话模式：

**业务范围模式**：选择一个业务范围发起对话。系统会自动加载该范围内所有智能体的能力、知识库和工具链。范围内的智能体可以相互委派任务（Sub-Agent Delegation）。

**单智能体模式**：直接选择一个特定的智能体进行一对一对话。

### 8.2 对话功能

- **流式输出**：AI 回复实时逐字显示
- **多轮上下文**：系统自动保持对话上下文
- **会话管理**：可以创建新会话、切换历史会话、恢复之前的对话
- **文件上传**：支持在对话中上传文件供智能体分析
- **Markdown 渲染**：AI 回复支持 Markdown 格式，包括代码高亮
- **Sub-Agent 委派**：在业务范围模式下，主智能体可以自动调用其他智能体协助完成任务

### 8.3 Chat 即 Mini-SaaS

在对话过程中，你可以用自然语言要求智能体生成一个小型应用（Mini-SaaS）。生成的应用可以发布到组织内部的应用市场，供其他成员直接使用。

## 九、Workflow（工作流）

### 9.1 工作流概述

工作流是将多个智能体按业务流程编排协作的自动化引擎。通过可视化 DAG（有向无环图）编辑器，你可以拖拽式构建复杂的业务流程。

### 9.2 工作流编辑器

编辑器界面分为三个区域：

- **左侧**：工作流列表，按业务范围分组，支持折叠
- **中央**：可视化画布，拖拽节点和连线
- **右侧**：Copilot 面板 + 节点属性编辑器

顶部有业务范围标签页，切换标签页可查看不同范围下的工作流。

### 9.3 节点类型

| 节点类型 | 说明 |
| --- | --- |
| Start | 工作流起点，可定义输入变量 |
| Agent | 调用一个智能体执行任务 |
| Action | 执行一个具体操作 |
| Condition | 条件分支，根据条件走不同路径 |
| Document | 文档处理节点 |
| Code Artifact | 代码执行节点 |
| Human Approval | 人工审批节点，暂停等待人工确认 |
| Loop | 循环节点 |
| Parallel | 并行执行节点 |
| End | 工作流终点 |

### 9.4 Workflow Copilot（工作流助手）

编辑器右侧的 Copilot 面板支持：

- **自然语言生成**：用自然语言描述业务流程，AI 自动生成完整的工作流节点和连线
- **自然语言修改**：对已有工作流用自然语言描述修改意图，AI 自动调整
- **多轮迭代**：支持多轮对话逐步完善工作流
- **执行日志**：运行工作流时，Copilot 面板实时显示执行进度和结果

### 9.5 运行工作流

点击工具栏的「运行」按钮执行工作流。如果 Start 节点定义了输入变量，系统会弹出表单让你填写运行时参数。

执行过程中：
- 画布上的节点会实时显示执行状态（等待、执行中、完成、失败）
- Copilot 面板显示详细的执行日志
- 支持中途终止执行

### 9.6 Checkpoint（检查点）

工作流支持在关键节点设置检查点，实现异步暂停/恢复：
- 人工审批：暂停等待管理者确认
- 表单输入：暂停等待用户补充信息
- 外部回调：暂停等待外部系统响应

### 9.7 版本管理

每个工作流支持多版本管理：
- 创建新版本时可基于现有版本
- 可标记某个版本为「Official」（正式版）
- 版本选择器在编辑器顶部，可快速切换

### 9.8 工作流导入

支持从图片导入工作流：上传一张流程图截图，AI 会自动识别并生成对应的工作流结构。

### 9.9 执行历史

点击工具栏的「History」按钮查看工作流的执行历史记录，包括：
- 每次执行的状态（成功/失败）
- 执行时间和耗时
- 节点级别的执行详情

## 十、触发机制

工作流支持三种触发方式：

### 10.1 手动触发

在工作流编辑器中点击「运行」按钮手动执行。

### 10.2 Webhook 触发

为工作流创建 Webhook URL，外部系统可以通过 HTTP POST 请求触发执行。

配置项包括：
- Webhook URL（自动生成）
- Secret（用于请求签名验证）
- IP 白名单
- 超时设置

点击工具栏的 Webhook 图标打开配置面板。

### 10.3 定时触发（Schedule）

使用 Cron 表达式设置定时执行计划。

配置项包括：
- Cron 表达式
- 时区
- 启用/禁用开关
- 运行时变量

点击工具栏的日历图标打开配置面板。

## 十一、Skills（技能）

### 11.1 技能概述

技能是智能体的能力扩展包。每个技能封装了一组特定的工具或 API 调用能力，可以被装备到智能体或业务范围上。

### 11.2 技能市场

系统提供两个技能来源：

**公共技能市场**：来自 skills.sh 的开源技能库，可以搜索、查看详情并一键安装。

**企业内部技能市场**：组织内部发布的自定义技能，支持投票和评分。

### 11.3 技能绑定

技能支持两层绑定：

- **Scope 级绑定**：将技能绑定到业务范围，该范围内所有智能体自动获得此技能
- **Agent 级绑定**：将技能绑定到特定智能体，仅该智能体可用

### 11.4 从 API Spec 创建技能

支持上传 OpenAPI/Swagger 规范文件，系统自动将 API 端点转换为可用的技能包。

### 11.5 从对话中发布技能

在 Chat 对话过程中，智能体创建的有价值的工具或脚本可以直接发布为企业内部技能，供其他智能体复用。

## 十二、MCP 集成

### 12.1 什么是 MCP

MCP（Model Context Protocol）是一种标准化的工具连接协议，让智能体能够调用外部系统的能力。Super Agent 支持 40+ 外部工具连接器。

### 12.2 配置 MCP 服务器

在业务范围的配置中添加 MCP 服务器：

- 填写服务器名称和描述
- 配置连接参数（URL、认证信息等）
- 设置 OAuth 密钥（如需要）
- 自定义请求头
- 启用/禁用状态

配置完成后，该业务范围内的智能体在对话和工作流执行时会自动获得 MCP 工具的调用能力。

### 12.3 支持的集成

常见的集成包括但不限于：Salesforce、Jira、Slack、GitHub、Google Workspace、企业微信、钉钉、飞书等。

## 十三、Knowledge（知识库）

### 13.1 文档组（Document Group）

知识库以文档组为单位组织。每个文档组是一个逻辑集合，可以包含多个文档文件。

创建文档组后，将其关联到业务范围，该范围内的智能体在对话时会自动检索相关知识。

### 13.2 上传文档

支持上传多种格式的文档（PDF、Word 等）。上传后系统会自动处理文档，建立向量索引用于 RAG 检索。

文档状态包括：处理中（processing）和已完成（completed）。

### 13.3 RAG 检索

当用户与智能体对话时，系统会自动从关联的知识库中检索相关内容，作为上下文提供给 AI 模型，确保回答基于企业自有知识。

## 十四、IM 渠道集成

### 14.1 支持的平台

Super Agent 支持将智能体接入以下即时通讯平台：

| 平台 | 说明 |
| --- | --- |
| Slack | 通过 Bot Token 接入 |
| Discord | 通过 Bot Token 接入 |
| Telegram | 通过 Bot Token 接入 |
| 钉钉（DingTalk） | 通过 Webhook 接入 |
| 飞书（Feishu） | 通过 App 接入 |
| 企业微信（WeChat Work） | 通过 App 接入 |

### 14.2 配置渠道

在业务范围的 IM 渠道管理中：

1. 选择平台类型
2. 填写 Bot Token 或 Webhook 配置
3. 绑定到目标业务范围
4. 启用渠道

配置完成后，用户可以直接在对应的 IM 平台中与智能体对话，消息会自动路由到正确的业务范围。

## 十五、Apps（应用市场）

### 15.1 应用市场概述

应用市场是组织内部的 Mini-SaaS 分发平台。智能体在对话中生成的应用可以发布到市场，供所有成员使用。

### 15.2 浏览应用

应用市场页面提供：

- **搜索**：按名称、描述、标签搜索
- **分类筛选**：tool、dashboard、form、game、utility 等类别
- **排序**：按热度、最新、评分、名称排序
- **视图切换**：网格视图和列表视图
- **趋势推荐**：展示最热门的应用
- **最近使用**：快速访问最近用过的应用
- **收藏夹**：收藏常用应用

### 15.3 运行应用

点击应用卡片上的「Run」按钮即可启动应用。应用在独立的运行环境中执行，支持数据持久化。

### 15.4 应用评分

使用过的应用可以进行星级评分和评价，帮助其他成员发现优质应用。

## 十六、Settings（系统设置）

### 16.1 Members（成员管理）

管理组织成员：
- 查看当前成员列表和角色
- 邀请新成员（通过邮件发送邀请链接）
- 修改成员角色
- 移除成员

邀请链接有有效期限制，过期后需重新发送。

### 16.2 Organization（组织设置）

Owner 可以管理：
- 组织名称
- 组织 Slug（URL 标识）
- 计划类型

### 16.3 API Keys

为程序化访问创建 API Key：
- 生成新的 API Key
- 设置权限范围
- 设置过期时间
- 查看最后使用时间
- 启用/禁用 Key

API Key 可用于通过 REST API 与 Super Agent 集成。

### 16.4 Appearance（外观）

切换界面主题：浅色模式、深色模式、跟随系统。

## 十七、Webhook 与 API 集成

### 17.1 Webhook

为工作流创建 Webhook 端点，允许外部系统触发工作流执行：

- 每个 Webhook 有唯一的 URL
- 支持 Secret 签名验证
- 支持 IP 白名单
- 自动记录调用历史（请求/响应）

### 17.2 REST API

Super Agent 提供完整的 REST API，覆盖所有功能模块：

| 模块 | 端点前缀 |
| --- | --- |
| 认证 | `/api/auth` |
| 智能体 | `/api/agents` |
| 工作流 | `/api/workflows` |
| 对话 | `/api/chat` |
| 技能 | `/api/skills` |
| 业务范围 | `/api/business-scopes` |
| 文档 | `/api/documents` |
| MCP | `/api/mcp` |
| 应用 | `/api/apps` |
| Webhook | `/api/webhooks` |
| 定时任务 | `/api/schedules` |
| API Key | `/api/api-keys` |
| 组织 | `/api/organizations` |

所有 API 请求需要在 Header 中携带 `Authorization: Bearer <token>` 进行认证。

## 十八、可观测性与审计

### 18.1 Agent 活动追踪

系统自动记录智能体的所有活动事件：
- Sub-Agent 调用
- 技能使用
- 工具调用
- 错误事件

事件数据按日聚合为指标汇总，可在智能体配置页面查看活动趋势。

### 18.2 工作流执行审计

每次工作流执行全程留痕：
- 整体执行状态和耗时
- 每个节点的状态流转（等待 → 执行中 → 完成/失败）
- 节点级别的输入输出数据

### 18.3 Langfuse 集成（可选）

如果配置了 Langfuse，系统会自动记录：
- 完整的 Agent 推理过程
- 工具调用链
- Sub-Agent 委派记录
- Token 使用量

支持按会话、用户、Agent 维度回溯，做到每一步都可解释。

## 十九、典型使用流程

以下是一个从零开始使用 Super Agent 的典型流程：

1. **创建业务范围**：例如创建「销售」范围，描述销售团队的业务职责
2. **上传知识库**：将销售手册、产品文档、FAQ 等上传到该范围的知识库
3. **创建智能体**：创建「销售助理」智能体，设置角色为销售支持，编写系统提示词
4. **安装技能**：为智能体安装 CRM 集成技能、邮件发送技能等
5. **配置 MCP**：连接 Salesforce、邮件系统等外部工具
6. **测试对话**：在 Chat 中与智能体对话，验证其能力
7. **构建工作流**：创建「新线索跟进」工作流，编排多个智能体协作
8. **设置触发**：为工作流配置 Webhook 触发，当 CRM 中有新线索时自动执行
9. **绑定 IM 渠道**：将智能体接入 Slack，让销售团队可以直接在 Slack 中提问
10. **持续优化**：查看记忆体和简报，根据反馈优化智能体配置

## 二十、常见问题

**Q：一个业务范围内可以有多少个智能体？**
A：没有硬性限制。建议根据实际业务需要创建，每个智能体负责一个明确的职能。

**Q：智能体之间如何协作？**
A：在业务范围模式的 Chat 中，主智能体可以自动委派任务给同范围内的其他智能体（Sub-Agent Delegation）。在 Workflow 中，通过 Agent 节点显式编排协作流程。

**Q：知识库支持哪些文档格式？**
A：支持 PDF、Word 等常见文档格式。上传后系统自动处理并建立向量索引。

**Q：工作流执行失败了怎么办？**
A：在工作流编辑器的 History 面板中查看执行详情，定位失败的节点和错误信息。修复后可以重新运行。

**Q：如何确保数据安全？**
A：所有数据按组织严格隔离。EC2 不开放 SSH 端口，通过 SSM Session Manager 安全访问。数据库加密存储，S3 启用服务端加密。IM 渠道的 Bot Token 加密保存。

**Q：支持哪些 AI 模型？**
A：当前通过 Amazon Bedrock 使用 Anthropic Claude 系列模型，默认使用 Claude Haiku 4.5。可在配置中切换为 Claude Sonnet 4 等其他模型。
